# AI 职场生存 — Reigns 式左右滑卡牌游戏设计

## 概述

一款以 AI 时代职场生存为主题的左右滑卡牌游戏，灵感来自 Reigns（王权）。玩家选择职业后，通过左滑/右滑应对 30 张职场事件卡牌，在 4 项属性之间保持平衡。属性归零或满值均导致 Game Over。

**目标受众：** 打工人/职场人，尤其是关注 AI 影响的群体
**游戏时长：** 2-4 分钟一局
**核心体验：** 简单上瘾、不费脑、有职场共鸣、适合截图分享
**技术栈：** React 19 + Vite + TypeScript，网页端 SPA

---

## 游戏流程

```
首页 → 选择职业（5选1）→ 滑卡牌（30张）→ 结局页
 /         /select           /game          /result
```

4 个路由，与现有项目结构一致。

## 核心机制

### 属性系统

4 项属性，范围 0-100：

| 属性 | 图标 | 含义 | 归零结局 | 满值结局 |
|------|------|------|----------|----------|
| 安全 (Safety) | 🛡 | 职位稳定性 | 被 AI 替代，失业 | 过度求稳，错失所有机会 |
| 技能 (Skill) | ⚡ | 专业能力 | 能力退化，无法胜任 | 过度内卷，身心俱疲 |
| 财务 (Finance) | 💰 | 经济状况 | 破产，负债累累 | 贪婪冒进，全部亏损 |
| 人脉 (Network) | 🤝 | 人际关系 | 被孤立，无人帮助 | 社交过度，精力耗尽 |

**Game Over 条件：**
- 任何属性 ≤ 0：对应属性的"归零结局"描述
- 任何属性 ≥ 100：对应属性的"满值结局"描述
- 存活 30 张卡：胜利

### 卡牌滑动

- **左滑** = 拒绝 / 忽略 / 保守选择
- **右滑** = 接受 / 行动 / 冒险选择
- 每个方向有不同的属性效果（可正可负，影响 1-3 个属性）
- 滑动前可预览：左右两侧显示受影响的属性图标和方向（↑/↓），不显示精确数值。数据层存储精确数值用于计算，UI 层仅展示图标方向

### AI 时代递进

卡牌分为 3 个阶段池，每阶段抽 10 张：

| 阶段 | 年份 | AI 发展 | 事件特点 |
|------|------|---------|----------|
| 早期 | 1-10 | AI 初现 | 温和事件，属性波动小（±5~10） |
| 中期 | 11-20 | AI 爆发 | 冲突加剧，属性波动大（±10~20） |
| 后期 | 21-30 | AI 全面渗透 | 高风险高回报，几乎每张都是生存挑战（±15~25） |

每阶段从该阶段卡池中随机抽 10 张，保证每局组合不同。

## 角色系统

6 个常驻角色，每张卡由一个角色"说话"：

| 角色 | 图标 | 身份 | 说话风格 |
|------|------|------|----------|
| 直属老板 | 👨‍💼 | 部门主管 | 命令式、绩效导向 |
| 卷王同事 | 👩‍💻 | 同组同事 | 竞争、炫耀、暗示 |
| 老前辈 | 🧓 | 资深员工 | 经验之谈、抵触变化 |
| AI 系统 | 🤖 | 公司AI助手 | 冷淡、效率导向 |
| 猎头 | 💼 | 外部猎头 | 诱惑、信息差 |
| HR | 📢 | 人力资源 | 官方通知、政策宣导 |

角色出现频率按阶段变化：
- 早期：老板和老前辈较多
- 中期：卷王同事和 AI 系统增多
- 后期：HR 和猎头频率上升

## 卡牌内容设计

### 每张卡的数据结构

```typescript
interface SwipeCard {
  id: string;
  character: CharacterType;  // 'boss' | 'colleague' | 'senior' | 'ai' | 'headhunter' | 'hr'
  text: { zh: string; en: string };  // 角色说的一句话（< 30字）
  leftChoice: {
    label: { zh: string; en: string };  // 左滑选项文案（< 8字）
    effects: Partial<Attributes>;
  };
  rightChoice: {
    label: { zh: string; en: string };  // 右滑选项文案（< 8字）
    effects: Partial<Attributes>;
  };
  stage: 1 | 2 | 3;  // 所属阶段
  careerIds?: string[];  // 仅特定职业出现（可选）
}
```

### 卡牌示例

**早期（第1-10年）：**

| 角色 | 台词 | 左滑 | 右滑 |
|------|------|------|------|
| 👨‍💼 老板 | "部门要引入 AI 辅助工具，你来负责试用？" | 推脱 (🛡+5 ⚡-5) | 接受 (⚡+10 🛡-5) |
| 👩‍💻 同事 | "我用 ChatGPT 一小时写完了周报，你要学吗？" | 不用 (🛡+5) | 教我 (⚡+8 🤝+5) |
| 🧓 前辈 | "AI 这东西就是泡沫，别浪费时间。" | 听劝 (🛡+5 ⚡-8) | 不信 (⚡+5 🤝-5) |

**中期（第11-20年）：**

| 角色 | 台词 | 左滑 | 右滑 |
|------|------|------|------|
| 📢 HR | "公司决定精简 20% 岗位，你的组也在名单上。" | 观望 (🛡-15 💰+5) | 主动转岗 (⚡+10 🛡-5 🤝-10) |
| 🤖 AI | "检测到你的工作效率低于 AI 基准线 40%。" | 忽略 (🛡-20) | 加班追赶 (⚡+15 🛡+10 💰-5) |
| 💼 猎头 | "AI 独角兽公司高薪招人，但要放弃现在的稳定。" | 留下 (🛡+10 💰-5) | 跳槽 (💰+20 🛡-15 🤝-10) |

**后期（第21-30年）：**

| 角色 | 台词 | 左滑 | 右滑 |
|------|------|------|------|
| 👨‍💼 老板 | "以后所有代码必须由 AI 生成，你来做审核。" | 抵制 (🛡-25 🤝+10) | 适应 (⚡-15 🛡+15) |
| 🤖 AI | "我可以完全替代你的岗位了。要合作还是竞争？" | 竞争 (⚡+20 🛡-20) | 合作 (🛡+15 ⚡-10 💰+10) |
| 📢 HR | "恭喜，你是部门唯一的'人类员工'了。" | 辞职 (💰-20 🤝+15) | 坚守 (🛡+20 ⚡+10 🤝-15) |

### MVP 卡牌数量

| 阶段 | 通用卡 | 职业专属卡 | 小计 |
|------|--------|-----------|------|
| 早期 (1-10) | 15 | 5×5=25 | 40 |
| 中期 (11-20) | 15 | 5×3=15 | 30 |
| 后期 (21-30) | 15 | 5×3=15 | 30 |
| **总计** | **45** | **55** | **100** |

每局从对应阶段池抽 10 张（混合通用 + 职业卡），保证不重复。

## UI 设计

### 首页
- 标题："AI 职场生存" / "AI Career Survival"
- 副标题："在 AI 时代，你能撑多久？"
- 一个"开始"按钮 → 进入职业选择

### 职业选择页
- 5 张职业卡横向排列或可滑动
- 每张卡：图标 + 职业名 + 一句话描述 + 4 属性预览条
- 点击选中 → 确认按钮

### 游戏主界面
- **顶部：** 4 个属性条（横向排列），实时动画更新
  - 属性 < 20 或 > 80 时闪烁警告
  - 属性变化时显示 +/- 数字飘字动画
- **中间：** 年份指示器（"第 X 年 / 30"）
- **主体：** 卡牌区域
  - 角色头像（圆形）+ 身份名
  - 场景台词（大字）
  - 左右两侧淡显：选项文案 + 属性影响方向
  - 支持触摸滑动和鼠标拖拽
  - 卡牌跟随手指/鼠标偏移旋转
  - 释放时飞出屏幕 + 下一张卡入场动画
- **底部：** 左右箭头提示（首次游戏显示教程）

### 结局页（暖白卡片风）
- 暖白渐变背景
- "你的 AI 时代人设是" + 标签名
- 存活年份 / 30
- 4 项最终属性（数字 + 进度条）
- 结局描述文案（AI 替代/失业/胜利等，非"死因"）
- "再来一局" + "分享" 按钮
- 分享生成图片（复用 PosterGenerator 思路）

### 动画要求
- 卡牌拖拽跟手：translateX + rotate（motion library）
- 属性条变化：弹性过渡 + 飘字（+10 / -15）
- 属性危险警告：脉冲闪烁
- 卡牌入场：从底部弹入 + 轻微弹跳
- Game Over：属性条爆裂/归零动画 → 淡出到结局页

### 结局标签（约 10 个）

根据存活年份 + 最终属性 + 归零/满值属性决定：

| 标签 | 条件 | 描述 |
|------|------|------|
| 🏆 AI 时代弄潮儿 | 通关且所有属性 > 50 | "你完美驾驭了 AI 时代！" |
| 🧘 佛系生存大师 | 通关，安全最高 | "稳如老狗，低调存活" |
| 📈 技术极客 | 通关，技能最高 | "靠硬实力杀出血路" |
| 🤝 人脉之王 | 通关，人脉最高 | "不是你会什么，是你认识谁" |
| 💰 财务自由 | 通关，财务最高 | "钱到位了，AI 随便替代" |
| 📦 被优化的打工人 | 安全归零 | "你的岗位已被 AI 全面替代" |
| 🔥 卷到燃尽 | 技能满值 | "你赢了所有人，但输给了自己" |
| 💸 创业炮灰 | 财务归零 | "All in AI，血本无归" |

- "你的岗位已被 AI 全面替代，HR 通知你明天不用来了。"
- "过度投资 AI 项目，账户余额归零。"
- "你达到了技能天花板，但身体和精神都撑不住了。"
- "恭喜你在 AI 浪潮中存活了 30 年！"

## 技术架构

### 与现有项目的关系

这是对现有项目的**重构**，保留：
- 技术栈（React 19 + Vite + TypeScript + Motion + CSS Modules）
- 路由结构（/, /select, /game, /result）
- i18n 双语系统
- 部分属性计算逻辑思路

替换：
- 游戏引擎（从年度事件选择 → 卡牌滑动）
- 数据模型（从 GameEvent/EventOption → SwipeCard）
- 全部 UI 组件（滑动卡牌是全新交互）
- 游戏状态管理

### 新数据模型

```typescript
// 角色类型
type CharacterType = 'boss' | 'colleague' | 'senior' | 'ai' | 'headhunter' | 'hr';

// 滑动卡牌
interface SwipeCard {
  id: string;
  character: CharacterType;
  text: { zh: string; en: string };
  leftChoice: {
    label: { zh: string; en: string };
    effects: Partial<Attributes>;
  };
  rightChoice: {
    label: { zh: string; en: string };
    effects: Partial<Attributes>;
  };
  stage: 1 | 2 | 3;
  careerIds?: string[];
}

// 角色定义
interface Character {
  type: CharacterType;
  name: { zh: string; en: string };
  icon: string;
}

// 结局标签
interface EndingTag {
  id: string;
  name: { zh: string; en: string };
  description: { zh: string; en: string };
  condition: (state: GameState) => boolean;
  priority: number;
}

// 游戏状态
interface GameState {
  careerId: string;
  currentCard: number;  // 0-29
  attributes: Attributes;
  history: SwipeHistory[];
  isGameOver: boolean;
  gameOverReason?: 'attr_zero' | 'attr_max' | 'won';
  gameOverAttr?: keyof Attributes;
  seed: number;
  deck: SwipeCard[];  // 本局的30张卡
}

// 滑动历史
interface SwipeHistory {
  cardIndex: number;
  cardId: string;
  direction: 'left' | 'right';
  attributesBefore: Attributes;
  attributesAfter: Attributes;
}
```

### 核心引擎

```
src/engine/
  types.ts          — 上述类型定义
  deckBuilder.ts    — 根据职业+种子从卡池构建30张牌组
  swipeEngine.ts    — applySwipe(), checkGameOver(), buildResult()
  random.ts         — 复用现有种子RNG
```

### 组件结构

```
src/components/
  SwipeCard/         — 可拖拽的卡牌组件（核心交互）
  AttributeBar/      — 单个属性条（带警告动画）
  AttributePanel/    — 4个属性条面板
  CareerCard/        — 职业选择卡片（简化版）
  EndingCard/        — 结局标签展示

src/pages/
  HomePage/          — 首页
  CareerSelectPage/  — 职业选择
  GamePage/          — 滑动游戏主页
  ResultPage/        — 结局页
```

### 滑动交互实现

使用 motion library 的拖拽功能：
- `drag="x"` 限制水平拖拽
- `dragConstraints` 限制拖拽范围
- `onDragEnd` 判断方向（偏移 > 阈值 = 确认选择）
- `animate` 飞出动画
- `whileDrag` 实时旋转倾斜
